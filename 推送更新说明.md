# 推送更新到 GitHub 说明

## ✅ 已检测到 Git 仓库

你的项目已经是一个 Git 仓库，并且已经配置了远程仓库：
- **远程地址**: `https://github.com/hypepsi/TrendRadar.git`
- **分支**: `master`

## 🚀 推送更新的方法

### 方法一：使用命令行（推荐）

如果你的系统已安装 Git，打开 PowerShell 或 CMD，在项目目录执行：

```bash
# 1. 查看更改状态
git status

# 2. 添加所有更改
git add .

# 3. 提交更改
git commit -m "更新：添加中英文新闻源，增强翻译功能（标题+摘要）"

# 4. 推送到 GitHub
git push origin master
```

**如果提示需要认证**：
- 使用 **Personal Access Token** 代替密码
- 生成地址：https://github.com/settings/tokens
- 权限选择：`repo`（完整仓库访问权限）

### 方法二：使用 GitHub Desktop

1. 打开 GitHub Desktop
2. 如果项目已打开，会显示更改的文件
3. 在左下角输入提交信息：`更新：添加中英文新闻源，增强翻译功能（标题+摘要）`
4. 点击 "Commit to master"
5. 点击 "Push origin" 推送到 GitHub

### 方法三：使用 VS Code

1. 在 VS Code 中打开项目
2. 点击左侧源代码管理图标（或按 `Ctrl+Shift+G`）
3. 在 "消息" 框中输入：`更新：添加中英文新闻源，增强翻译功能（标题+摘要）`
4. 点击 "提交" 按钮（或按 `Ctrl+Enter`）
5. 点击 "同步更改" 或 "推送到" 按钮

### 方法四：使用批处理脚本

直接双击运行 `推送更新.bat` 文件（如果已安装 Git）

## ⚙️ 推送后的操作

### 1. 检查 GitHub Actions

1. 进入 https://github.com/hypepsi/TrendRadar
2. 点击 **Actions** 标签
3. 查看是否有新的 workflow run
4. 可以手动触发：点击 "Hot News Crawler" → "Run workflow"

### 2. 检查 GitHub Pages

1. 进入 **Settings** → **Pages**
2. 确认已启用 GitHub Pages
3. 访问你的网站：`https://hypepsi.github.io/TrendRadar/`

### 3. 配置 Secrets（如果需要推送通知）

1. 进入 **Settings** → **Secrets and variables** → **Actions**
2. 添加需要的 Secrets：
   - `FEISHU_WEBHOOK_URL` - 飞书机器人
   - `WEWORK_WEBHOOK_URL` - 企业微信
   - `EMAIL_FROM` - 发件邮箱
   - `EMAIL_PASSWORD` - 邮箱密码/授权码
   - `EMAIL_TO` - 收件邮箱
   - 等等...

## 🔍 常见问题

### Q: 推送时提示需要认证？

A: GitHub 已不再支持密码认证，需要使用 Personal Access Token：
1. 访问：https://github.com/settings/tokens
2. 点击 "Generate new token (classic)"
3. 选择权限：`repo`
4. 生成后复制 token
5. 推送时，用户名输入你的 GitHub 用户名，密码输入 token

### Q: 如何查看推送是否成功？

A: 
1. 访问 https://github.com/hypepsi/TrendRadar
2. 查看最新的 commit 记录
3. 检查文件是否已更新

### Q: GitHub Actions 没有自动运行？

A:
1. 检查 `.github/workflows/crawler.yml` 文件是否存在
2. 手动触发一次：Actions → Hot News Crawler → Run workflow
3. 检查是否有错误日志

## 🎉 完成！

推送成功后，你的更改就会在 GitHub 上生效，GitHub Actions 会自动运行爬虫，GitHub Pages 会自动更新网站。
